Facebook's Prophet forecasting library

Mar 5, 2017 #sysid #ml #machine learning #deep learning

Be a prophet for 1960’s airline data

Facebook released a forecasting library for business timeseries, called Prophet. It’s designed to work similar to the phantastic scikit-learn library and is therefor easy to use.

In order to give an example of its usefulness and effectiveness I will apply it to the good old airline passenger dataset.

The input to Prophet is always a dataframe with two columns: ds and y. The ds (datestamp) column must contain a date or datetime (either is fine). The y column must be numeric, and represents the measurement we wish to forecast.

df = pd.read_csv(DATA_HOME_DIR+'/international-airline-passengers.csv',
                 sep=';',
                 names=['ds', 'y'],
                 header=0,
                 parse_dates=[0],
                 nrows=144,
                )

Since the data seems to incorporate an exponential growth trend, we preprocess it by taking the log and use the linear trend fitting capabilities of Prophet.

df['y'] = np.log(df['y'])

We only have monthly data, so certainly there will be no weekly seasonality in the date. Forecasting must take this into account and choose the right frequency for the target dates.

m = Prophet(weekly_seasonality=False)

Now we can simply call the fit method in order to fit the data.

m.fit(df)

Predictions are then made on a dataframe with a column ds containing the dates for which a prediction is to be made. You can get a suitable dataframe that extends into the future a specified number of days using the helper method Prophet.make_future_dataframe. By default it will also include the dates from the history, so we will see the model fit as well.

future = m.make_future_dataframe(periods=36, freq='M')

The predict method will assign each row in future a predicted value which it names yhat. If you pass in historical dates, it will provide an in-sample fit. The forecast object here is a new dataframe that includes a column yhat with the forecast, as well as columns for components and uncertainty intervals.

forecast = m.predict(future)

The result is promising:

Prophet automatically gives you an overall trend analysis and decomposes the time series into its constituing compontents like yearly seasonality:

Seeing a almost perfect linear trend confirms the original hypothesis of having a exponential growth trend.

The steps in the yearly trend plot seem to be an artefact of fitting. Having a peak in July for travel numbers seem to indicate the holiday season in the western hemisphere. However this is just an asumption since I have not been born then and are not familiar of the holiday season arrangments of the time.
